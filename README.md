# 雙 AI 協作起手包

讓 Claude 跟 ChatGPT（Codex）兩個 AI 在你的電腦上組成一個小工班：**一個當工頭、一個當師傅、你只驗收成品**。

這份起手包由一位非工程師的餐飲業老闆實測整理——他用這套配置跑真實營運工作，你拿到的是已經踩過雷、修過坑的版本。

## 這是什麼

大多數人用 AI 的方式是「一個很強的 AI 包辦所有事」。這套的玩法不一樣：

- **Claude 當工頭**：負責想清楚、把任務切成明確的工單（含驗收條件、禁區、停損）、驗收成品、仲裁分歧
- **Codex 當師傅**：接工單施工——寫 code、生圖、做文件，只做工單內的事
- **兩個 AI 之間用「交接單」溝通**：換一個對話、換一天，工作記錄都在，不失憶
- **你是老闆**：講需求、看成品、做最後決定

最關鍵的一步是「CLI 直派」：**Claude 可以直接指揮 Codex 幹活**，你不用在兩個視窗之間當傳聲筒複製貼上。

## 它能做到什麼

1. **你講一句需求，兩個 AI 自己分工把東西做出來**——例如「幫這篇貼文配一張圖」，工頭開單、師傅生圖、工頭驗收，你只看最後那張圖
2. **防止 AI「自稱做完了」**——工頭驗收看的是實物（檔案真的存在、圖真的長對、測試真的過），不信口頭報告
3. **防止越改越大、收不了尾**——工單寫死「只准碰哪些檔案、不准順手重構、碰到什麼狀況立刻停」
4. **額度花在刀口上**——貴的模型只做判斷（開單、驗收、仲裁），粗活交給另一邊做，兩邊訂閱都省
5. **互相當第二意見**——一個 AI 卡關兩天的問題，換一個不同出身的 AI 看，常常一輪就通；審查方只能報問題、不能亂改，由工頭查證定案
6. **換對話不失憶**——完成的定義寫在工單和交接單裡（檔案），不靠 AI 的記憶

## 實際案例

這些都是這套配置真實跑出來的結果（發生在一家餡餅店老闆的電腦上）：

**案例一：一句話換一張商用配圖。**
老闆說「幫我的 FB 貼文配圖」。Claude 讀貼文、開工單（指定角色設定圖、風格、只准存某資料夾、只產一張、生不出來就停）→ 直接派給 Codex → Codex 用內建生圖完成 → Claude 打開圖逐項驗收（人物一致、無錯字、風格正確）→ 交付。老闆全程只講了一句話。

**案例二：兩個 AI 值不同班、用交接單接力。**
晚上 Codex 做完一份簡報的視覺改版，寫好交接單投進「交接箱」資料夾；隔天 Claude 開工自動掃箱，把成果歸檔、更新戰情板。兩個 AI 從沒同時在線，工作照樣接得起來。

**案例三：誠實回報勝過假完成。**
一次測試中工單要求 Codex 寫一份報告檔，但當時它處在唯讀模式寫不了。它的回覆是「此工作區唯讀，我沒有假裝已寫入」——這正是這套制度要的：**做不到就說做不到**，而不是編一個看起來完成的假象。

## 怎麼開始（三步）

1. 打開你已經有的那個 AI（Claude 或 ChatGPT，網頁版就行）
2. 把本 repo 的 [`雙AI協作起手包_SETUP.md`](./雙AI協作起手包_SETUP.md) 丟給它（或直接貼這個 repo 網址），說：
   > 照這份手冊帶我一步一步做，我做完你檢查，過了才走下一步。
3. 它說什麼你做什麼。手冊分五個 Phase、每關有檢查點，從補訂閱、安裝、蓋制度、打通直派、到跑出第一個實驗成品，大約一個下午。

## 誠實聲明

- 需要 Claude 和 ChatGPT **兩邊的付費訂閱**（各自入門級即可，撞到額度再升）
- 手冊以 Windows 為主（Mac 有註記但未完整實測）
- AI 工具改版很快，內容若與現況不符，直接把錯誤丟給你的 AI，它通常能自己繞過去
