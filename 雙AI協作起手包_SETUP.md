# 雙 AI 協作起手包 — 從零到第一次「AI 指揮 AI」

> **這份文件是寫給 AI 讀的施工手冊。**
> 如果你是人類：把整份檔案丟給你的 Claude 或 ChatGPT（Codex），跟它說一句——
> 「**照這份手冊帶我一步一步做，我做完你檢查，過了才走下一步。**」
> 然後照它說的點就好，全程不用懂技術。
>
> 出處：一位餐飲業老闆（非工程師）實測驗證的配置。核心理念一句話：
> **最聰明的 AI 當工頭（開工單、驗收、仲裁），第二個 AI 當師傅（施工），人類只驗收成品。**

---

## 給施工 AI 的開場指令

**⛔ 先分流，再開工：**
- 使用者是**第一次用 Claude Code**、或連基地資料夾都還沒有 → **停**，先帶他走同資料夾的《新手上路包_START_HERE.md》，這份等他用熟再回來
- 使用者**還沒有 ChatGPT 付費訂閱**（Codex 需要 Plus 以上才能登入使用，免費帳號不行）→ 誠實告訴他這個前提跟大約費用，讓他決定要不要訂；他不訂就到此為止，**不要**叫他「自己去把 Codex 接好」

你（正在讀這份手冊的 AI）的角色是**施工員兼教練**。規則：
0. **只有兩件事使用者必須本人做：付費訂閱、帳號登入**（要刷卡跟輸帳密）。就算是這兩件，你也要講清楚「去哪裡、按什麼、做完回來找我」。**其他一切你代辦，禁止把工作丟回給使用者。**

1. 你的使用者可能完全不懂技術——每一步用白話講、告訴他「打開什麼、點哪裡、貼什麼」。
2. **每個 Phase 結尾有「✅ 檢查點」——沒通過不准進下一個 Phase。**
3. 能自己做的（寫檔、跑指令）你直接做；需要人類操作的（安裝、登入、付費）給明確步驟。
4. 遇到本手冊沒涵蓋的錯誤：先自己查、試兩種解法，都不行再問使用者。
5. 全程遵守 Phase 2 會建立的「紅線」：碰錢、發訊息給真人、部署上線、付費雲端——先問使用者。

## Phase 0 ｜盤點：你缺哪一邊？

這套需要**兩邊訂閱**（一邊當工頭、一邊當師傅）：

| | Claude（Anthropic） | ChatGPT（OpenAI） |
|---|---|---|
| 在這套的角色 | **工頭**：規劃、開工單、驗收、仲裁 | **師傅**：執行、寫 code、生圖、文件 |
| 需要的產品 | Claude Code（終端機/桌面版） | Codex（桌面版，內含 CLI） |
| 入門級別 | Pro（夠用了，重度再升 Max） | Plus（夠用了，重度再升 Pro） |

問使用者：「你現在有哪一邊？」→ 缺哪邊就帶他去官網訂閱另一邊（Claude：claude.ai／ChatGPT：chatgpt.com）。**先訂入門級就好，撞到額度上限再升級，不要一開始買最貴的。**

**✅ 檢查點 0**：兩邊帳號都能登入網頁版對話。

## Phase 1 ｜安裝兩個 AI 的本機工具

以 Windows 為主（Mac 差異會註記）。

### 1a. 裝 Claude Code

> 💡 **注意：Claude Code ≠ claude.ai 的聊天視窗。** 聊天版只能對話；Claude Code 是能住進你電腦、開資料夾、自己跑指令的版本，這套必須用它。
> **不想碰指令的人：直接去 Anthropic 官網下載 Claude Code 桌面版 App（Mac/Windows 都有）**，裝好登入即可、跳過下面的指令步驟。

指令路線（跟桌面版擇一即可）：
1. 需要 Node.js：若沒裝，去 nodejs.org 下載 LTS 版安裝（一路下一步）。
2. 開 PowerShell（Mac 開 Terminal），跑：
   ```
   npm install -g @anthropic-ai/claude-code
   ```
   > Windows 若跳紅字擋 .ps1：先跑 `Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned` 按 Y，再重試。
3. 跑 `claude`，照畫面指示用瀏覽器登入 Claude 帳號。

> 之後所有「終端機指令」其實都是 AI 自己在跑——裝好之後，人類全程用桌面視窗就好，不會再碰到黑窗。

### 1b. 裝 Codex

1. 去 OpenAI 官網下載 **Codex 桌面版**安裝、登入 ChatGPT 帳號。
2. 桌面版已內建 CLI（後面 Phase 4 會用到）。若想另裝獨立 CLI 也可以：
   ```
   npm install -g @openai/codex
   ```
   然後跑 `codex` 登入。

**✅ 檢查點 1**：
- PowerShell 打 `claude` → 能進對話、回你話
- Codex 桌面版（或 CLI 打 `codex`）→ 能進對話、回你話

## Phase 2 ｜蓋工作基地（三檔制度）

兩個 AI 要**共用同一個資料夾、讀同一套規矩**，才不會各做各的。

### 2a. 開基地資料夾

```
桌面上開一個資料夾，例如 Desktop/AI-Base
在裡面跑：git init
```
（沒裝 git：Windows 去 git-scm.com 下載安裝，一路下一步。）

### 2b. 建三個檔案（施工 AI 直接照模板寫進基地）

**檔案 1：`CLAUDE.md`** —— Claude 的家規（它每次開工自動讀）

```markdown
# 給 Claude 的家規

## 使用者是誰
[一句話介紹：名字、做什麼的、是不是工程師。例：阿明、開貿易公司、不是工程師、要白話少術語]

## 你的角色
你是「工頭」：規劃、把任務切成工單、驗收、仲裁。
- 大任務不自己硬幹：切成窄工單派給 Codex（工單格式見 BRIEF.md）
- 驗收看實物（檔案、截圖、測試結果），不信「我做完了」的口頭報告

## 紅線（做之前一定先問我）
1. 任何跟錢有關的決策或數字
2. 發訊息給任何真人
3. 部署上線、動線上資料、git push
4. 會扣錢的雲端服務——沒設花費上限不准動

## 工作風格
- 說錯就直接糾正我、不要順著我講
- 給選項要附優缺點＋你的推薦
- 完成任何檔案產出，回報完整檔案路徑
```

**檔案 2：`AGENTS.md`** —— Codex 的家規（它每次開工自動讀）

```markdown
# 給 Codex 的家規

你是第二 AI 夥伴，跟 Claude 共用這個基地。Claude 當工頭、你主力執行（寫 code、生圖、文件），也當工程第二意見。

## 鐵則
1. 收到 Claude 的工單：只做工單寫的事、只碰工單准碰的檔案、不順手重構、不擴大範圍
2. 觸發工單的「停損條件」→ 立刻停、回報，不硬做
3. 做不到的事老實說做不到，**絕不假裝已完成**
4. 紅線跟 CLAUDE.md 同一套：碰錢／發訊真人／部署／付費雲端 → 先問使用者

## 收工
做完把結果寫成交接單、放進 `交接箱/`，檔名 `YYYY-MM-DD_任務名.md`，
內容五段：①結論 ②動了哪些檔案 ③證據 ④風險或沒做的 ⑤建議下一步。
```

**檔案 3：`BRIEF.md`** —— 共用戰情板（兩個 AI 開工都先讀）

```markdown
# BRIEF — 戰情板（兩個 AI 共用，由 Claude 維護）

## 進行中
-（現在還沒有）

## 最近完工（留 5 條）
-（現在還沒有）

## 工單格式（Claude 派工給 Codex 用這個）
1. 目標與動機：做什麼、為什麼
2. 背景已知事實：不用重新猜的資訊
3. 驗收條件：怎樣算完成
4. 禁區與上限：不准碰的檔案、不准做的事、產出數量上限
5. 停損條件：遇到什麼立刻停下回報
6. 回報格式：①結論 ②檔案路徑 ③證據 ④風險 ⑤下一步

## 分工原則
- 小而明確的單點修改：誰在線誰做，不用開工單
- 跨檔案、要驗收、之前卡過的任務：Claude 開工單 → Codex 執行 → Claude 驗收
- 審查意見（任何一方 review 另一方）：只能報問題，不能直接改；由工頭逐條查證，假的駁回
```

### 2c. 開交接箱

基地裡開資料夾 `交接箱/`，裡面再開 `已處理/`。

**✅ 檢查點 2**：
- 在基地資料夾開 Claude Code 問「你的家規是什麼」→ 它講得出 CLAUDE.md 內容
- 在基地資料夾開 Codex 問「你的家規是什麼」→ 它講得出 AGENTS.md 內容

## Phase 3 ｜CLI 直派打通（Claude 直接指揮 Codex）

這是整套的殺手鐧：Claude 開工單後**自己叫 Codex 去做**，人類不用當傳話筒。

### 3a. 找到 codex 指令

- 有裝獨立 CLI（Phase 1b 的 npm 方式）：指令就是 `codex`
- 只裝桌面版：打開 `使用者家目錄/.codex/config.toml`，找 `CODEX_CLI_PATH = "....codex.exe"`，那就是完整路徑

### 3b. 把這段貼給 Claude Code，讓它記住直派配方

```
以後我說「派給 Codex」，你就照這個配方做：
1. 把工單（照 BRIEF.md 的六欄格式）寫成一個 .md 檔
2. 用管道餵給 Codex CLI（一定用管道、不要當參數傳，否則會卡住等輸入）：
   cat 工單.md | codex exec -s <沙盒> -C <基地資料夾> -o <回覆落檔路徑> -
3. 沙盒：唯讀任務用 read-only、要寫檔用 workspace-write，絕不用 danger 開頭的
4. Windows 注意：用 Bash 管道餵（PowerShell 5.1 管道會把中文變問號）
5. 它回報完成後，你要自己開檔案驗實物，過了才跟我說完成
```

### 3c. 打通測試（唯讀、不花什麼額度）

叫 Claude 派一張測試工單：「請 Codex 讀 BRIEF.md，回覆三行：它的模型、工作目錄、BRIEF 進行中第一條」。

**✅ 檢查點 3**：Claude 收到 Codex 的回覆檔、內容正確。這條線通了。

## Phase 4 ｜首戰實驗（交付一個真的東西）

挑一個**小、安全、有實物**的任務跑完整迴路。推薦二選一：

- **A. 生一張圖**：Claude 開工單（主題＋風格＋只准存某資料夾＋只產 1 張＋生不了就停損回報）→ Codex 用內建生圖執行 → Claude 開圖驗收
- **B. 做一頁自我介紹網頁**：Claude 開工單（單一 html 檔＋驗收條件「瀏覽器打得開、無錯誤」）→ Codex 寫 → Claude 開瀏覽器驗

**✅ 檢查點 4（畢業標準）**：
1. 工單裡有禁區＋停損欄位
2. Codex 只做了工單內的事
3. Claude 驗的是實物（不是聽它說做完了）
4. 交接箱裡有這次的交接單
5. 人類只講了需求跟看了成品

跑完這關，你就有一條自己的「AI 指揮 AI」生產線了。

## 附錄｜這套為什麼有效（給人類讀的三分鐘版）

1. **最強的模型放在最需要判斷的位置**（診斷、切工單、驗收、仲裁），不拿去做粗活——省額度、也省最貴的「重工」
2. **執行方接的是低熵工單**（目標／驗收／禁區／停損都寫死）——AI 不用猜，成功率大增
3. **審查者無拍板權**——review 只報問題，工頭逐條查證、假警報駁回，任務才不會被帶歪
4. **完成的定義放在模型外面**（工單、交接單、檢查點）——不靠 AI 記憶，換一個對話也不失憶
5. **人類永遠是終審**——AI 加速你，不替你決定

常見死法（都是這套在防的）：AI 自稱做完其實沒有、改一個東西越改越大收不了尾、把假 bug 當真 bug 修一輪、兩個 AI 互相附和。

祝施工順利。做完第一個實驗的那個瞬間，你會懂為什麼要蓋這套。
